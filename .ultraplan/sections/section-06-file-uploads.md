# Section 06: File Uploads (MinIO)

## Overview
Enable file attachments on tasks using self-hosted MinIO. Files upload directly from the browser to MinIO via presigned URLs generated by the Rust backend. File metadata is stored in PostgreSQL alongside the task.

## Risk: [green] - MinIO presigned URL pattern is well-documented

## Dependencies
- Depends on: 04
- Blocks: 11, 12
- Parallel batch: 3

## TDD Test Stubs
- Test: Presigned upload URL is generated with correct bucket and expiry
- Test: File metadata is saved to database after successful upload
- Test: File download URL is generated and accessible
- Test: Files larger than 10MB are rejected before upload
- Test: Deleting a task attachment removes metadata from database
- Test: User must be a board member to upload files
- Test: Presigned URLs use the public-facing MinIO URL, not internal Docker hostname
