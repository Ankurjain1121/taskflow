Implementation Plan

Phase 1: Fix Broken Dark Mode CRITICAL

Step 1.1 — Fix Hardcoded Colors (10 component files)

Replace pattern:

bg-white          → bg-[var(--card)]
text-gray-900     → text-[var(--card-foreground)]
text-gray-500     → text-[var(--muted-foreground)]
border-gray-200   → border-[var(--border)]
hover:bg-gray-50  → hover:bg-[var(--muted)]
bg-indigo-50      → bg-indigo-50 dark:bg-indigo-900/30
text-indigo-600   → text-indigo-600 dark:text-indigo-400
border-indigo-200 → border-indigo-200 dark:border-indigo-800

Step 1.2 — Add Missing CSS Infrastructure

/* Surface elevation system */
:root {
  --surface-0: #F6F7FB;  --surface-1: #ffffff;
  --surface-2: #ffffff;  --surface-3: #ffffff;
}
html.dark {
  --surface-0: var(--background);
  --surface-1: var(--card);
  --surface-2: #3a3c58;  /* dropdowns, popovers */
  --surface-3: #42446a;  /* modals, dialogs */
}

/* Scrollbar */
html.dark *::-webkit-scrollbar-thumb { background: rgba(255,255,255,0.15); }
html.dark *::-webkit-scrollbar-track { background: var(--background); }

/* Form controls */
html.dark ::placeholder { color: var(--muted-foreground); }
html.dark input:disabled, html.dark textarea:disabled {
  background: var(--muted); color: var(--muted-foreground); opacity: 0.5;
}

Step 1.3 — PrimeNG Dark Overrides

Verify Aura dark mode handles: p-dialog, p-toast, p-tooltip, p-dropdown, p-inputtext. Add overrides where needed.

Phase 2: Add Multiple Themes NEW

Step 2.1 — Update ThemeService

export type Theme = 'light' | 'dark' | 'dracula' | 'dimmed' | 'system';
export type ResolvedTheme = 'light' | 'dark' | 'dracula' | 'dimmed';

// applyTheme():
// light   → no class
// dark    → class="dark"
// dracula → class="dark dracula"
// dimmed  → class="dark dimmed"
